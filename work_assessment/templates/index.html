Model-View-Controller is a software design pattern for developing web applications


Model

  • This part of the MVC represents actual data that an applications deals with.

  • Technically its the logical structure of an application's database, including the relationships and constraints that determine how data can be stored and accessed.

  • It doesn't depend on either view or controller, meaning any manipulation done by view and/or controller doesn't affect the model.

View

  • This part presents what the user will see on their user interface.

  • The data created in the model or rather, user input is viewed and entered here before it is stored in its respective model.

Controller

  • This part acts as the 'middle-man' between the model and the view.

  • It updates the model when users manipulate the view and updates the view when the model changes.

  • It basically controls the interaction between the model and the view.
